## 实验一 生产者-消费者问题实现

实验类型：设计类
实验学时：2学时
实验目的：熟悉临界资源、信号量操作的定义与物理意义；了解进程通信的方法；掌握进程互斥与进程同步的相关知识；掌握用信号量机制解决进程之间的同步与互斥问题；通过生产者-消费者问题，深刻理解进程同步问题。
实验内容与要求：
内容：实现经典同步问题：生产者-消费者问题   
要求：
一个大小为10的缓冲区，初始状态为空；
2个生产者，随机等待一段时间，往缓冲区中添加数据，若缓冲区已满，等待消费者取走数据后再添加，重复10次
2个消费者，随机等待一段时间，从缓冲区中读取数据，若缓冲区为空，等待生产者添加数据后再读取，重复10次
实验报告
实验题目
程序中使用的数据结构及其说明
源程序（需要加功能函数注释，与关键代码行注释）
程序运行结果截图及说明
观察算法运行结果，分析算法中使用的进程间通信的方法。



## 实验二 进程调度算法的设计

实验类型：设计类
实验学时：2学时
实验目的：通过对进程调度算法的设计，深入理解进程调度的原理
实验内容与要求：
内容：实现短进程优先调度算法（SPF）和时间片轮转调度算法（RR）或者最短剩余时间优先算法（SRTF）和HRRN（高优先级优先调度算法）
要求：
进程通过定义一个进程控制块的数据结构（PCB）来表示；
每个进程需要赋予进程ID、进程到达时间、进程需要运行的总时间、阻塞时间的属性；
在RR中，以1为时间片单位；
运行时，输入5个进程序列，按照进程的ID输出其执行序列。

这里只给出一个基础PCB的定义，可以根据实际需要实现算法的需要，在此基础上进行结构体成员添加（不允许减少）。
enum Status{running, ready, blocked}
enum Policy{spf,rr,srtf,hrrn}
typedef struct PCB {
int id, cputime, alltime, startblock, blocktime;
Status state;      //进程状态
struct PCB * next; // 链指针
}
实验报告
实验题目
程序中使用的数据结构及其说明
源程序（需要加功能函数注释，与关键代码行注释）
程序运行结果截图及说明
观察算法运行结果，比较模拟的两种算法的优缺点。



## 实验三 银行家算法

实验类型：设计类
实验学时：2学时
实验目的：银行家算法是避免死锁的代表性算法，本实验旨在加深了解有关动态资源申请、避免死锁、安全状态等概念，并体会和运用避免死锁的具体实施方法，自动设计模拟程序。
实验内容与要求：
内容：银行家算法模拟
要求：
系统能接收自定义进程数目、资源类型和每种类型资源的数目；
可输入每个进程对每种资源的最大需求、已经获得的数量；
当某进程发起某种资源请求时，计算系统状态是否安全。

实验报告
实验题目
程序中使用的数据结构及其说明
源程序（需要加功能函数注释，与关键代码行注释）
程序运行结果截图及说明
观察算法运行结果，分析系统资源分配策略（即按何种策略生成安全序列）



## 实验四 内存页面置换算法的设计

实验类型：设计类
实验学时：2学时
实验目的：Linux系统中，为了提高内存利用率，提供了内外存进程对换机制，内存空间的分配和回收均以页为单位进行，一个进程只需部分装入便可运行，并且支持请求调页的存储管理方式。本实验要求学生通过请求分页存储管理中页面置换算法模拟，了解虚拟存储技术的特点，并掌握主要的请求分页存储管理中的页面置换算法。
实验内容与要求：
内容：实现先进先出（FIFO）页面转换算法、最近最久未使用（LRU）页面置换算法、OPT算法
要求：
系统采用固定分配、局部置换的策略
分配给进程的物理块数在程序中可选：3或者4
系统随机产生页面访问序列地址（如12300，37987，54230等），序列长度为20，
运行时，先输入分配给进程的物理块数（3或者4），系统输出随机页访问序列、物理块装入页面情况，并最终打印出缺页次数与缺页率

实验报告
实验题目
程序中使用的数据结构及其说明
源程序（需要加功能函数注释，与关键代码行注释）
程序运行结果截图及说明
观察算法运行结果，比较模拟的三种算法的优缺点。
